# Java 핵심 개념 정리

## 1. 업캐스팅 (Parent A = new Child())

### 핵심 개념

부모 타입으로 자식 객체를 참조하는 것을 업캐스팅이라고 한다.

```java
Parent A = new Child();
```

### 동작 원리

* 실제 생성된 객체: Child
* 참조 타입: Parent

즉, 객체는 자식이지만 부모 타입의 시선으로 바라보는 구조이다.

### 필드와 메서드 차이

#### 필드 (변수)

* 부모 타입 기준으로 접근
* 오버라이딩 개념 없음

```java
A.parentField  // 가능
A.childField   // 불가능
```

#### 메서드

* 오버라이딩된 경우 → 자식 메서드 실행
* 오버라이딩 안 된 경우 → 부모 메서드 실행

```java
A.method();
```

👉 실행 기준은 "실제 객체(Child)"

## 2. static (정적 변수)

### 핵심 개념

static 변수는 객체가 아니라 클래스에 속한다.

```java
class Test {
    static int count = 0;
}
```

### 특징

* 객체마다 따로 존재하지 않음
* 클래스 하나당 하나만 존재
* 모든 객체가 공유

### 동작 방식

```java
Test a = new Test();
Test b = new Test();

a.count++;
```

👉 결과

```plaintext
b.count도 같이 증가됨
```

또는

```java
Test.count++;
```

👉 동일하게 증가됨

### 핵심 정리

* 객체로 접근해도 결국 같은 static 변수
* 클래스 자체에 저장됨

## 3. 증감 연산자 (++ 연산자)

### b++ (후위 증가)

```java
int b = 1;
int result = b++;
```

### 동작 순서

1. 현재 값 사용 (result = 1)
2. 이후 증가 (b = 2)

👉 "지금은 안 바뀐 것처럼 보이고, 다음에 반영"

---

### ++b (전위 증가)

```java
int b = 1;
int result = ++b;
```

### 동작 순서

1. 먼저 증가 (b = 2)
2. 값 사용 (result = 2)

👉 "지금 바로 반영"

---

## 최종 한방 정리

### 업캐스팅

* 필드 → 부모 기준
* 메서드 → 실제 객체(자식) 기준
근데 메서드 안 변수는 자식 기준 부모 필드 사용 한다고 메서드안에서 사용 안함
메서드안 호출은 자기 클래스가 아니라 오버라이딩 된거 

### static

* 클래스 단위로 하나만 존재
* 모든 객체가 공유

### 증감 연산자

* b++ → 나중 증가
* ++b → 먼저 증가
